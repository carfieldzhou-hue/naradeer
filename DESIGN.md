## 〇、设计支柱（Design Pillars）

四条不可妥协的玩家体验，所有设计决策都以此为衡量标准：

1. **治愈感优先（Healing First）**
   奈良喂鹿的核心情绪是"被小动物需要、被信任、被治愈"。任何增加焦虑、惩罚、挫败的机制都要克制。失败要轻、反馈要暖、奖励要足。

2. **鞠躬即高潮（Bow Is the Moment）**
   鹿鞠躬讨食是这个游戏最核心的"心动瞬间"。它是 fantasy hook（真实奈良鹿会鞠躬），也是情绪价值锚点。鞠躬动画必须好看、喂食反馈必须丰盛——这是玩家记住这个游戏的理由。

3. **传播即生存（Share to Survive）**
   作为网页游戏，传播链是命脉。分享不是附加功能，而是被 baked into 核心经济与元进度。但"鼓励分享"不能等于"无限刷钱"——必须守住经济稀缺。

4. **渐进解锁、永远有新东西（Progressive Reveal）**
   每一关都要给玩家"新能力 / 新鹿 / 新道具"中至少一个。避免长尾关卡重复感。宁可内容少而解锁节奏密，也不要内容多却一次倒完。

## 一、核心循环（Core Loop）

### 1.1 Moment-to-Moment（0–30 秒）
- **动作**：玩家用摇杆/WASD 移动，靠近鹿 → 鹿进入检测范围、主动接近 → 在 1.2 距离停下鞠躬（重复前鞠循环）→ 玩家点击/按 E 喂仙贝
- **反馈**：爱心粒子爆发 + 撒花 + 音效 + 鹿开心跳跃 + 头部点头 + HUD 闪光
- **奖励**：进度 +1、图鉴收录、2.8s 后鹿开心再爆一波粒子

> **设计意图**：这一层必须"零门槛、高反馈"。玩家不需要瞄准/时机（休闲定位），但反馈要密集到每一次喂食都像"被治愈一次"。鞠躬动画 + 粒子爆发是情绪价值的核心载体。

### 1.2 Session Loop（5–30 分钟 / 单关）
- **目标**：喂够 N 只鹿通关（L1=5 → L10=16）
- **张力**：仙贝只有 3 个起步，必须用金钱（100円/个）向摊贩购买；金钱来自宝箱/散落金币/摇钱树/分享
- **决策**：钱花在哪——买仙贝 vs 买道具 vs 攒着；走哪条路找鹿；是否绕开水池
- **解决**：喂够通关 → 通关结算 → 下一关

> **设计意图**：单关内要有"资源紧张 → 做取舍 → 解决"的完整弧线。难度来源是"仙贝供给"（摊贩递减、金币递减），不是"操作难度"。

### 1.3 Long-Term Loop（小时–周）
- **进度**：L1→L10 关卡推进 + 鹿稀有度解锁（普通 L1 / 稀有 L2 / 珍稀 L3 / 传说 L4）+ 道具渐进解锁（L2/L3/L5/L7）
- **收集**：16 只鹿图鉴，含金色鹿、蝴蝶鹿等特殊皮肤
- **留存钩子**：称号系统（按累计分享次数解锁，存 localStorage 跨会话保留）

> **设计意图**：长线靠"收集 + 解锁 + 称号"三重驱动。称号跨会话保留是关键——玩家回来不是为了"重玩"，是为了"离下一个称号还差几次分享"。

## 二、经济系统（Economy）

### 2.1 经济流向总览

```
[来源]                              [池]        [出口]
宝箱(递减) ─┐
散落金币(递减)─┼──→  金钱(跨关继承) ──→ 买仙贝(100円/个)
摇钱树(L3+) ─┘                        └──→ 道具商店(6种,渐进解锁)
分享(+100円/次,递减) ────────────────────┘
```

### 2.2 设计原则
- **金钱跨关继承**：减少难度断层，存量随关数累积。仅 restart 回 L1 时清零。
- **仙贝不继承**（每关重置 3 个）：这是难度的主要来源——高关摊贩稀疏（L8 仅 1 个），必须靠"买/分享换钱"维持仙贝供给。
- **递减曲线**：宝箱池 `1000 × 0.6^(L-1)`（L2≈600，平缓衰减，下限 50）；散落金币、宝箱数、摊贩数均随关递减。难度靠"资源变稀缺"而非"操作变难"。

### 2.3 分享经济（传播钩子）
- 每次成功分享 +100 円，4 秒冷却，**收益递减**（防刷）：同关多次分享收益递减，守住经济稀缺。
- 累计分享次数推进称号元进度（见 §6），跨会话保留。
- **设计红线**：分享是"鼓励"而非"必需"。不能出现"不分享就过不去"的设计——分享换的是便利，不是通行证。

### 2.4 道具商店（经济出口）⭐
**问题**：原经济只有单一出口（买仙贝），钱花不出去 → 零消费决策 → 经济深度不足。
**对策**：6 种道具作为新增出口，按关卡渐进解锁，每个都是独立的消费决策。

| 道具 | 解锁 | 价格 | 效果 | 设计意图 |
|------|------|------|------|----------|
| 🎵 鹿笛 | L2 | 150 | 召唤附近3只未喂鹿靠近 | 解决"找不到鹿"痛点，把盲走变成主动召唤 |
| 📡 寻鹿雷达 | L2 | 120 | 30秒箭头指向最近未喂鹿 | 同上，导航辅助，减少空跑 |
| 💨 速度符 | L3 | 200 | 20秒移速×1.8 | 缩短高关赶路时间，缓解"地图过大" |
| ☂️ 避水符 | L3 | 180 | 60秒免疫水池 | 把纯惩罚变成"可付费规避"的风险管理 |
| 👘 隐身斗篷 | L5 | 300 | 15秒稀有/传说鹿不逃跑 | 降难靠近高稀有鹿，服务收集目标 |
| 🚲 环保自行车 | L7 | 500 | 本关移速×2.5（整关,一次性） | 高关长线加速，整关投资型消费 |

**解锁节奏**：L2 解锁 2 个（鹿笛+雷达，解决找鹿）、L3 解锁 2 个（速度+避水，解决赶路与风险）、L5 解锁 1 个（隐身，服务稀有收集）、L7 解锁 1 个（自行车，高关整关加速）。**绝不全倒**——每关都有"新东西可买"的期待。

**平衡注记**：速度符(×1.8)与自行车(×2.5)不叠加（自行车优先），避免出现 ×4.5 的失控速度。所有计时器每关重置（自行车亦然）。价格均为 `[PLACEHOLDER]`，需 playtest 调整。

## 三、鹿 AI 与性格设计（Deer AI）

### 3.1 五种性格（涌现趣味的来源）
| 性格 | 行为 | 设计意图 |
|------|------|----------|
| 温顺 Gentle | 主动靠近、喂后会跟随3秒 | 给玩家"被信任"的治愈感 |
| 害羞 Shy | 靠近就逃、玩家移动快会惊跑 | 制造"要轻手轻脚"的张力 |
| 好奇 Curious | 中距离试探性靠近 | 半推半就的趣味 |
| 高冷 Aloof | 很近才反应、然后远离 | 难钓的鱼，增加成就感 |
| 暴躁 Aggressive | 警告→冲撞→抢仙贝 | "故事时刻"，玩家会谈论的危险遭遇 |

> **设计意图**：性格差异让"喂鹿"这个重复动作每次都不一样——这是对抗核心循环单调性的第一道防线。emergent（涌现）趣味来自"你不知道靠近的鹿会怎样"。

### 3.2 稀有度系统
- 四级：Common / Uncommon / Rare / Legendary，按 `minLevel` 门槛解锁（L1/L2/L3/L4）。
- **越稀有越难接近**：检测范围递减（传说仅 40%），惊跑阈值递减，游走范围递增。这让传说鹿真的有"传说感"。
- 视觉差异化：发光（蓝/紫/金）、金色鹿、蝴蝶鹿翅膀。收集图鉴驱动。
- 隐身斗篷道具可抵消稀有鹿的惊跑，服务"靠近传说鹿"的收集目标。

### 3.3 调戏式生气（v1.1 改动）
- **旧设计（已废弃）**：玩家没仙贝靠近鹿 → 鹿生气撞人。问题：惩罚玩家执行游戏自己的经济（每关只给3个仙贝，必然频繁0仙贝）。
- **新设计**：鹿鞠躬期待食物（`expectingFood`）后，玩家**走开不喂**才生气。两阶段提示：开始离开 → warn toast；真走开 → angry toast。
- **设计意图**：生气是"玩家选择"的后果（你选择调戏），不是"游戏经济"的副作用。提示确保玩家理解规则。

### 3.4 鞠躬动画（v1.1 改动，最高优先级）
- **旧问题**：Bow 状态无鞠躬动画，头部停在后歪的 rest pose。
- **新设计**：Bow 状态下头部做缓慢深沉的向前鞠躬循环（`bowPhase` 驱动，baseline -0.08 + dip -0.42，rotation.z 负=前），像真奈良鹿讨食点头。
- **喂食反馈强化**：3 颗爱心 + 撒花爆发；2.8s 后开心再补爱心+撒花。
- **设计意图**：鞠躬是游戏的核心心动瞬间（支柱2），必须好看且反馈丰盛。

## 四、关卡与难度曲线

### 4.1 难度来源（按设计意图排序）
1. **仙贝供给收紧**：摊贩 `max(1, 9-L)`、金币池递减 → 后期"买不起仙贝"
2. **地图扩张**：`0.5 × 2^(min(L,7)-1)`，L7 封顶 → 后期"找不到鹿、赶路久"
3. **水池障碍**：L2+ 出现，递增至 8 个 → 导航风险
4. **鹿稀有度门槛**：高稀有鹿难接近 → 收集难度

### 4.2 已知问题与对策
| 问题 | 严重度 | 状态 | 对策 |
|------|--------|------|------|
| 高关地图过大(走路模拟器) | P0 | 部分缓解 | 道具(鹿笛/雷达/速度符/自行车)缩短赶路；后续可下调封顶尺寸 |
| 分享刷钱(无限源) | P0 | 待修 | 计划：同关分享收益递减（见 §2.3） |
| L7–L10 无区分度 | P1 | 待修 | 计划：新机制(天气/季节/鹿王遭遇) |
| 无失败状态 | P1 | 暂缓 | 用户决定先不加倒计时；用"资源紧张"软性施压 |

> **设计哲学**：难度靠"资源稀缺"而非"操作惩罚"。这是休闲治愈定位决定的——玩家不该因手残而失败，而应因"没规划好仙贝/道具"而感到挑战。

## 五、Onboarding（新手引导）

### 5.1 现状
- 开场文字提示（桌面/移动分别显示对应操作）
- L2+ 进关 toast 提示水池
- **缺口**：无引导式首喂、无箭头指摊贩、首次可能因0仙贝被撞（v1.1后改为调戏才撞，已缓解）

### 5.2 目标（待实现）
- [ ] 核心动词（喂食）在 30 秒内引入
- [ ] 首次成功有保障——箭头指向最近摊贩→强制首购→箭头指向最近鹿→保证首喂
- [ ] 至少一个机制通过探索发现（非文字）
- [ ] 首局结束在钩子上（解锁/悬念/"再来一局"）

## 六、称号系统（元进度 / 留存钩子）

按**累计分享次数**解锁，存 `localStorage('naradeer_total_shares')`，跨会话保留。

| 累计分享 | 称号 | 说明 |
|----------|------|------|
| 0 | 新手饲鹿人 | 初来奈良公园 |
| ≥1 | 初露锋芒 | 完成第一次分享 |
| ≥3 | 分享小能手 | 累计分享 3 次 |
| ≥10 | 鹿群挚友 | 累计分享 10 次 |
| ≥25 | 奈良百事通 | 累计分享 25 次 |
| ≥50 | 鹿仙·传说分享王 | 累计分享 50 次 |

> **设计意图**：称号是"分享行为的长期记账"。单次分享给钱（短期激励），称号给身份认同（长期激励）。跨会话保留是关键——玩家回访的动机是"离下个称号还差几次"。

## 七、调参哲学（Tuning Philosophy）

1. **所有数值都是假设**：标 `[PLACEHOLDER]` 的需 playtest 验证。先有公式，再调魔法数。
2. **先定义"破"再 playtest**：知道失败长什么样（如"分享刷爆=经济破产"、"L6 走路超10分钟=枯燥"），才能识别问题。
3. **公式优于硬编码**：用 `0.6^(L-1)` 而非逐关填数，便于整体调曲线。
4. **调参入口集中**：见 `LEVEL_DESIGN.md` 第五节「调参速查」，改一处即可生效。
5. **分离观察与解读**：playtest 笔记先记"发生了什么"，再记"意味着什么"。

### 关键调参杠杆（影响最大的几个）
| 杠杆 | 位置 | 影响 |
|------|------|------|
| 金币池曲线 | `Game.ts` `moneyPool` | 整体经济松紧 |
| 摊贩数曲线 | `Game.ts` `createVendorsForLevel` | 仙贝供给（难度主源） |
| 地图缩放 | `Game.ts` `getBoundsForLevel` | 赶路时长（枯燥感） |
| 分享收益/冷却 | `Game.ts` `onShareSuccess` | 经济可解性 |
| 道具价格 | `Game.ts` `SHOP_ITEMS` | 消费决策深度 |
| 鹿检测范围 | `Deer.ts` `getDetectionRange` | 找鹿难易 |
