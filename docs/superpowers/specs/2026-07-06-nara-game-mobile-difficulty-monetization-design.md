# 奈良公园喂鹿游戏 — 手机适配·难度·变现 设计方案

## 概述

对现有 Three.js 奈良公园喂鹿游戏进行三项改造：**手机 UI 适配**、**逐关难度递增 + 鹿性格系统**、**非付费变现（广告+分享）**，同时增加**日式 BGM 与音效**提升沉浸感。

---

## 1. 手机 UI 适配

### 目标
一套代码，桌面/手机自适应，手机上触控舒适、信息清晰。

### 布局策略
- 使用 CSS 媒体查询 `@media (max-width: 768px)` 切换手机布局
- 桌面保留现有布局，手机端重新排列

| 元素 | 桌面 | 手机 |
|------|------|------|
| HUD 顶栏 | 三栏铺开（分数/关卡/倒计时） | 单行精简，只显示关键数字+图标 |
| 摇杆 | 右下，半径 50px | 左下，半径 60px，死区自适应 |
| 按钮(喂食/冲刺) | 右下小圆按钮 | 底部两个 ≥48px 大圆，拇指可及 |
| 提示(hint) | 居中浮动 | 底部固定半透明条 |
| 分享/广告按钮 | 右上角 | 底部导航栏集成 |
| 图鉴覆盖层 | 居中卡片 | 全屏卡片，更宽的内边距 |

### 触控交互增强
- **摇杆死区自适应**：轻触边缘 = 慢走，重推 = 快跑（映射 `move` 向量幅度）
- **点击喂食**：靠近鹿时点击喂食按钮（已有）
- **可选自动寻路**：点击地面目标位置，角色自动寻路过去（后续迭代）

### 文件变更
- `src/styles.css` — 新增 `@media (max-width: 768px)` 块，调整 HUD、摇杆、按钮、提示层
- `index.html` — 可能需要调整 DOM 结构以支持响应式
- `src/systems/Hud.ts` — update() 根据窗口尺寸切换显示模式

## 2. 难度递增体系

### 关卡难度解锁表

| 关卡 | 新增机制 | 具体规则 |
|------|----------|----------|
| L1 | 教学关 | 5只鹿，无时间限制，无惩罚，水温顺鹿为主 |
| L2 | 🏊 水域惩罚 | 走进池塘/河道→扣1块仙贝+弹回+水花音效；出现害羞鹿 |
| L3 | ⏱ 倒计时 | 屏幕顶部计时条，超时则闯关失败；出现高冷鹿 |
| L4 | 🦌 鹿更难接近 | 稀有/传说鹿警觉距离再缩短30%，逃跑速度+50%；出现暴躁鹿 |
| L5+ | 🚧 障碍更多 + 综合 | 障碍物密度×2，以上所有机制叠加 |

### 水域惩罚详规

- **触发条件**：Player 位置进入 `Park.waterMeshes[]` 中任一 CircleGeometry 的半径内
- **惩罚逻辑**：
  1. 有仙贝 → `crackerCount--`，播放落水音效，红色闪烁提示
  2. 无仙贝 → `money = max(0, money - 50)`，播放落水音效
  3. 无仙贝无钱 → 只弹回，无额外惩罚
- **弹回**：沿 Player 到水面的方向向量推回 1.5 单位
- **跳跃豁免**：Player 跳跃中（`!isOnGround()`）进入水域不触发惩罚
- **冷却**：同一水域 2 秒内不重复触发

### 倒计时详规

- **L3+** 关卡开始后启动倒计时
- 时长 = `60 + (level - 1) * 15` 秒（L3=90s, L4=105s, ...），上限 180s
- UI：HUD 新增计时条，颜色从绿渐变到红
- 超时后：弹出"时间到"覆盖层
  - 可看广告续命 30 秒（每关 1 次）
  - 或重新开始本关

### 障碍物密度

- 当前：固定 40 个障碍物，`generateObstacles(40)`
- L5+：改为 `generateObstacles(40 + (level - 5) * 15)`，上限 100

## 3. 鹿性格系统

### 五种性格

每种鹿在生成时分配一个性格，影响其行为逻辑。

| 性格 | 权重 | 行为 | 出现关卡 |
|------|------|------|----------|
| **温顺** 😊 | 30% | 玩家进入 5m 范围内主动靠近到 2m；喂食判定范围 +50%；不逃跑 | L1 |
| **害羞** 😳 | 25% | 玩家进入 4m 范围后退保持 3m 距离；喂食判定范围 -20%；速度 0.6x 后退 | L2 |
| **好奇** 🧐 | 20% | 先靠近到 3m 观察，玩家一动就跑；喂食窗口短（2 秒内不喂就跑） | L1 |
| **高冷** 😐 | 15% | 不主动靠近，玩家靠近到 2.5m 就走开；喂食判定范围 -40%；需要堵路包抄 | L3 |
| **暴躁** 😠 | 10% | 玩家进入 3m → 头顶 ❗ 预警；进入 1.5m → 低头冲锋撞人；被撞扣 1 仙贝+弹飞 | L4 |

### 暴躁鹿行为状态机

```
IDLE → (玩家进入 3m) → WARNING (头顶❗, 2 秒)
WARNING → (玩家退出 5m) → IDLE
WARNING → (玩家进入 1.5m 或 2 秒后) → CHARGING (直线冲向玩家)
CHARGING → (撞到玩家 或 跑出 8m) → FLEEING (跑开 10m, 5 秒冷却)
FLEEING → (冷却结束) → IDLE
```

### 性格集成到现有 Deer 类

- `Deer.ts` 已有 `DeerRarity` 和 `DEER_NAMES` 等元数据
- 新增 `DeerPersonality` 枚举，`PERSONALITY_BY_INDEX` 数组（同 deer index 绑定，确定性）
- `Deer.update()` 根据 `DeerPersonality` 分支行为逻辑
- `getDeerInfo()` 返回新增 `personality` 字段 → Journal 卡片显示

## 4. 变现与分享

### 分享机制（修复 + 增强）

**当前问题**：`doShare()` 先试 `navigator.clipboard` 再试 `navigator.share`，顺序反了，且没有点击反馈。

**修复方案**：

```
click 分享按钮
  → 立即 toast "正在准备分享…"
  → navigator.share() 优先（手机原生分享面板）
    → 成功 → +100円，toast "分享成功！"
    → 取消/失败 → fallback 到剪贴板
  → 剪贴板 (navigator.clipboard.writeText)
    → 成功 → +100円
    → 失败 → document.execCommand('copy') 兜底
```

**分享增强**：
- 每关可分享 1 次，+100 円（保持现有频率）
- 分享文案包含关卡信息 + URL
- 后续可加每日首次分享额外奖励（localStorage 记录）

### 激励广告

真实广告 SDK 接入需要平台支持，这里先实现**模拟广告**流程，后续可替换为真实 SDK。

| 广告位 | 触发条件 | 流程 | 奖励 | 限制 |
|--------|----------|------|------|------|
| 看广告得钱 | 点击"📺 广告"按钮 | 显示 5 秒倒计时"广告播放中…" → 自动关闭 | +100 円 | 每关 2 次 |
| 看广告补仙贝 | 仙贝 = 0 时出现按钮 | 同上 | +2 块仙贝 | 每关 1 次 |
| 看广告续命 | 倒计时归零/L5+闯关失败 | 同上 | 继续 30 秒 | 每关 1 次 |

**UI 位置**：
- 手机：底部导航栏放一个"📺 广告"按钮（与分享并列）
- 桌面：右上角分享按钮旁

**模拟广告实现**：
- 点击 → 创建半透明覆盖层 "広告再生中… (5)" 倒计时
- 倒计时归零 → 覆盖层消失 → 发放奖励
- 用 `setTimeout` 模拟广告时长
- 预留真实 SDK 接口：`AdSystem.playRewardedAd(): Promise<boolean>`

## 5. 日式 BGM & 音效

### 原则
- 全部用 Web Audio API 程序化生成，不依赖外部音频文件
- 保持包体小，加载快

### BGM（背景音乐）

- **音阶**：日本五声音阶（宫商角徵羽）= `C D E G A`（都节音阶）
- **音色**：多个 Oscillator 叠加 + 滤波器
  - 主旋律：三角波 + 低通滤波（模拟尺八）
  - 伴奏：正弦波 + 颤音（模拟筝）
  - 节奏：低频噪声脉冲（模拟太鼓）
- **层级结构**：
  ```
  AudioContext
   ├─ MasterGain (0.3)
   │   ├─ BGM_Gain → BGM 节点
   │   ├─ SFX_Gain → 音效节点
   │   └─ Ambience_Gain → 环境音节点
  ```
- **关卡速度**：L1 BPM=60，每 5 关 +5 BPM，上限 80
- **循环**：16 小节的旋律 loop，无限重复

### 音效清单

| 场景 | 音色 | 实现 |
|------|------|------|
| 喂鹿成功 | 上升叮咚 | ✅ 已有 |
| 鹿高兴 | 三连上升音 | ✅ 已有 |
| 获得金币 | 硬币碰撞 `[short noise + high sine ping]` | 新增 |
| 购买仙贝 | 酥脆感 `[filtered noise burst]` | 新增 |
| 落水 | 扑通 `[low sine + splash noise]` | 新增 |
| 鹿发怒 | 太鼓 `[低频正弦波，快速衰减]` | 新增 |
| 倒计时紧迫 | 心跳 `[低频脉冲，越来越快]` | 新增 |
| 关卡胜利 | 和风鼓号 | 增强现有 |
| 摇钟 | 寺庙钟声 | ✅ 已有（需绑定到 TempleBell 交互） |
| 看广告 | 简短金币声 | 新增 |

### AudioSystem 扩展

`src/systems/AudioSystem.ts`：

- `startBGM(level: number)` — 启动关卡 BGM
- `setLevel(level: number)` — 更新 BGM 速度
- `stopBGM()` — 停止 BGM
- `splash()` — 落水音效
- `angryDeer()` — 鹿怒音效
- `coin()` — 金币音效
- `buyCracker()` — 购买音效
- `heartbeat(intensity: number)` — 倒计时紧迫音效
- `adComplete()` — 广告完成音效
- `templeBell()` — ✅ 已有
- `dispose()` — ✅ 已有（需同时停止 BGM）
